## ⚠️ 오류 검토 및 주의사항

### 1. 카테고리명 변경 시 주의사항

**문제점:**
- "대나무숲" 카테고리를 변경하면 기존 게시글과의 호환성 문제 발생 가능
- Firestore 쿼리에서 카테고리 필터링 시 기존 데이터와 불일치

**해결 방안:**
- **옵션 A (권장)**: 카테고리명은 유지, 브랜드명만 변경
  - "대나무숲" 게시판은 그대로 유지
  - 커뮤니티 전체 이름만 "널자"로 변경
  - 장점: 기존 데이터와 호환성 유지, 마이그레이션 불필요
  - 단점: 브랜드 일관성 약간 떨어짐

- **옵션 B**: 카테고리명도 변경
  - "대나무숲" → "널자" 또는 "널자 커뮤니티"
  - 기존 게시글의 카테고리 필드 업데이트 필요
  - Firestore 마이그레이션 스크립트 필요
  - 장점: 브랜드 일관성 높음
  - 단점: 마이그레이션 작업 필요, 잠재적 오류 가능성

**추천**: 옵션 A (카테고리명 유지)

### 2. 타입스크립트 오류 가능성

**확인 필요:**
- 카테고리 타입 정의가 하드코딩되어 있는지 확인
- `boardCategories` 배열의 타입 정의
- 카테고리 필터링 로직의 타입 안정성

**예상 오류:**
```typescript
// 만약 카테고리를 변경한다면
const boardCategories = [
  { value: '널자', label: '🗣️ 널자', emoji: '🗣️' },
  // ...
]
// 기존 게시글의 category가 '대나무숲'이면 필터링 실패 가능
```

**해결책:**
- 카테고리 변경 시 마이그레이션 스크립트 작성
- 또는 카테고리명 유지 (권장)

### 3. 빌드 오류 가능성

**확인 사항:**
- 메타데이터 변경 시 Next.js 빌드 오류 없음
- 문자열 리터럴 변경은 타입 오류 없음
- 다만, 카테고리 변경 시 쿼리 로직 확인 필요

### 4. SEO 및 소셜 미디어 영향

**변경 사항:**
- 메타데이터 title/description 변경
- Open Graph 태그 (필요시 추가)
- Twitter Card 태그 (필요시 추가)

**영향:**
- 기존 검색 결과와 불일치 가능
- 소셜 미디어 공유 시 새로운 정보 표시
- 검색 엔진 재인덱싱 필요 (시간 소요)

### 5. 사용자 경험 영향

**고려사항:**
- 기존 사용자에게 브랜드명 변경 안내 필요
- 로그인 페이지에서 변경된 브랜드명 확인 가능
- 기존 북마크/링크는 계속 작동 (URL 변경 없음)

**원인**: Firestore 쿼리에서 새로운 카테고리명으로 필터링하지만 기존 데이터는 이전 카테고리명 사용
